hexo+github搭建个人博客

2018-05-24

一、环境配置

1、node：一款流行的web服务器，也是一款服务器端语言

安装：官网（https://nodejs.org） 下载V8.11.2版本的msi安装包，默认配置安装
检测环境：
  ①win+R → cmd
  ②输入node -v （出现版本号则已安装；如果提示不是内部命令，则需安装）
目的：提供博客项目运行所需的web服务器环境

2、git：版本控制器

安装：官网(https://git-scm.com/download)下载最新版本，默认配置安装
检测环境：
  ①win+R → cmd
  ②输入git --version （出现版本号则已安装；如果提示不是内部命令，则需安装）
   或者在桌面右键菜单出现git bash here则代表安装成功

3、安装hexo博客框架(https://hexo.io/)

①win+R → cmd
②输入npm install hexo-cli -g    /*通过npm包管理器全局安装hexo脚手架*/
 npm:    node的包管理器（类似于应用商店），提供node工具包的安装和卸载(uninstall)
 install：    导入、安装
 cli：    脚手架
 -g：    (global)对面前命令的修饰，表示全局安装，在本机任何路径都可访问

二、博客搭建

1、初始化博客项目

①桌面右键选择git bash here(不一定在桌面，可在任意英文路径下)
②hexo init blog                /*blog为项目名称，名字随意，但必须是英文*/
③切换到 blog 路径                /*可用cd命令，或在blog文件夹内右键选择git bash here*/
④npm install                    /*安装项目依赖*/
⑤hexo server                    /*启动服务器；简写hexo s*/
⑥在浏览器地址栏输入以下地址以预览
 localhost:4000 或者 127.0.0.1:4000

blog路径下 _config文件改标题(改博客的) 注意空格的
blog\source_posts路径下 hello-world文件改内容和具体文件的标题

2、修改主题

去[hexo官网](https://hexo.io/)选择并安装大标题。

三、hexo项目托管到github

hexo常规操作常规

  • 清理项目缓存

何时使用？
修改了博客内容却没有效果或者修复了bug报错依然存在

  • 编译博客

何时使用？
每次hexo clean之后都需要重新编译
部分情况下修改了_config.yml配置文件后需要重新编译
手动新建博客文章之后需要重新编译

  • 重启服务器
    在之前启动服务的命令行窗口使用快捷键”ctrl+c”可以停止hexo服务
    然后紧接着输入hexo s启动
    何时使用？
    修改了_config.yml需要重启服务

三、hexo项目托管到github

(一)本机验证github的key

  1. 打开github官网 注册个人账号
    并记录下用域名和邮箱，并且验证邮箱
  2. 命令窗口输入：ssh-keygen -t rsa -C “邮箱” /生成key文件，一路回车即可/
    接下来依次打开：c:/Users/Administrator/.ssh路径
    找到：id_rsa.pub文件右键选择记事本打开，全选并复制里面的内容(key)
  3. 回到github页面，找到头像右侧倒三角下拉菜单，选择settings
  4. 在页面左侧选择ssh and gpk
  5. 点击右上角绿色 add ssh按钮
  6. 在表单输入title(随意)和key(粘贴之前复制到的key)；然后点击绿色create ssh按钮
  7. 命令窗口验证用户名和邮箱
    git config –global user.name “用户名”
    git config –global user.email “邮箱”
    如果验证成功，没有任何提示

    (二)创建github用户名同名仓库

    在github页面点击头像左侧的+，选择New repository
    重点：仓库名必须是:用户名.github.io,严格区分大小写，如：
    假如用户名为aBc，那么仓库名就为:aBc.github.io

    (三)项目推送

  8. 配置hexo
    找到项目主目录下的_config.yml文件
    在文档的最下方添加如下内容：

注意：用户名严格区分大小写

  1. 安装gcmw（解压压缩包，双击install.cmd）
  2. 打开博客项目根目录，右键git bash here
  3. 安装deployer-git依赖：
    npm install hexo-deployer-git –save
  4. 依次执行如下命令：
    hexo clean
    hexo g /重新编译/
    hexo d /推送/
    中途会弹出github的验证，输入相应的应户名密码就好
  5. 如果推送成功，在浏览器打开http://用户名.github.io即可访问

ps：

整个项目有2个_config.yml
根目录的_config.yml负责整个项目的一些配置，如：博客名称、语言等
themes文件夹内的_config.yml负责主题样式方面的配置

搭建评论系统：

https://valine.js.org/ → 注册 → 设置/应用Key →
复制APP ID和App Key 以修改themens/使用中的主题/config配置文件 中的对应内容(71行) →
70行on: true → notify: true → verify: true

标签：

①主题配置文件 menu：下，标签：tags/all
②所有博客文件 — 中 tags：下另起一行输入：- all

分类：

输入命令：hexo new page categories

硬推：

①git init /在要退的文件路径下，用git bush输入命令/
②git add 文件名 /添加文件/
③git commit -m “说明” /对添加的描述/
④git remote add origin 地址 /仓库地址，https传不了用ssh/
⑤git push -u origin master /推送/
git add 文件夹/ /添加文件夹/
git add . /添加路径下的所有文件/
.gitignore文档里为忽略的文件名

从github仓库下载：

方法1：
git pull –rebase origin master /把服务器上的文件pull下来/
方法2：
git clone 仓库地址 /文件路径下，用git bush输入命令/
